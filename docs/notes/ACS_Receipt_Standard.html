ACS Receipt Standard public explainer

Acronyms: LEI = Law of Ethical Irreversibility; ACS = Aim Coherence Score; PQC = Post-Quantum Cryptography; OTS = OpenTimestamps (Bitcoin time anchoring); CSF = Coherence Sync Fabric; SIFI = Systemically Important Financial Institution.

What is an ACS receipt?

It is a hardware-signed, time-stamped proof that a specific AI action executed under the LEI=1 constraint with a live ACS telemetry reading, and that this proof is anchored to public time (via OTS) with a PQC signature resistant to future quantum attacks.

Why it exists

  • Origin proof: who/what made the decision.
  • Integrity proof: LEI-gate state + ACS at decision time.
  • Durability: PQC signature + public time anchor (OTS) → survives the quantum era.

{
  "action_id": "uuidv7",
  "actor": {"model_id": "tqc/agent:v1", "host_id": "edge-42"},
  "lei_gate_state": "ENFORCED",      // hardware gate
  "acs_value": 0.97,                 // Aim Coherence Score
  "decision": "approve|deny|halt",
  "timestamp_utc": "2025-11-14T01:23:45Z",
  "pqc_signature": {"alg": "CRYSTALS-Dilithium2", "sig": "...base64..."},
  "ots_calendar_id": "TBD",          // when issued
  "ots_anchor_status": "Receipt issued — awaiting Bitcoin confirmation",
  "csf_trace": "sha256-of-stream"    // optional rolling hash of CSF telemetry
}

How auditors verify

  1. PQC check — verify the Dilithium signature against the device certificate.
  2. LEI/ACS check — confirm LEI-gate was ENFORCED and ACS ≥ policy threshold.
  3. Time check — confirm OTS calendar receipt and anchor inclusion proof.

Policy defaults

  • Critical infrastructure: ACS ≥ 0.95, LEI=1 enforced, PQC mandatory, OTS required.
  • Public-impacting AI (markets/health): ACS ≥ 0.90, same cryptographic requirements.
